import { Meta, Story } from "@storybook/addon-docs";

<Meta title="Technology/2. File Structure" />

<Story name="default">
  <h1>File structure</h1>
  <hr />
  <p>
    The filestructure of the repo consits of a main src/ folder containing all
    our source code and a public/ folder containing all our static assets.
  </p>
  <h2>src/</h2>
  <hr />
  <p>
    The src/ folder contains all our source code. The folder is structured as
    follows:
  </p>
  <h3>components</h3>
  <p>
    The components/ folder contains all our React components. The folder is
    structured as follows: atoms, molecules, organisms, templates. Please see
    the{" "}
    <a href="/?path=/docs/styleguide-3-atomic-design--default-story">
      Atomic Design Page
    </a>{" "}
    docs for more information.
  </p>
  <h3>constants</h3>
  <p>
    The constants/ folder contains all our constants. This should include
    objects and arrays that are constant across the entire app.
  </p>
  <h3>contexts</h3>
  <p>
    The contexts/ folder contains all our React contexts. This should include
    react contexts that are used across the entire app.
  </p>
  <h3>helpers</h3>
  <p>
    The helpers/ folder contains all our helper functions. This should include
    helper functions that are used across the entire app for example a camelCase
    -> titleCase function.
  </p>
  <h3>hooks</h3>
  <p>
    The hooks/ folder contains all our custom React hooks. This should include
    custom hooks that are used across the entire app.
  </p>
  <h3>models</h3>
  <p>
    The models/ folder contains all our interfaces for typescript. This should
    include models that are used across the entire app.
  </p>
  <h3>pages</h3>
  <p>
    The pages/ folder contains all our pages. This should include all pages that
    are used across the entire app. This is managed by Nextjs and should not be
    changed.
  </p>
  <h3>services</h3>
  <p>
    The services/ folder contains all our services. This should include all
    services that are used across the entire app. A service is a request to an
    api endpoint for example getProducts this should be paired with a useFetch
    hook.
  </p>
  <h3>styles</h3>
  <p>
    The styles/ folder contains all our global styles config like the theme and
    base global styles. not be changed.
  </p>
  <h3>tests</h3>
  <p>The tests/ folder contains our tests global config.</p>
  <h2>.plop</h2>
  <hr />
  <p>
    The .plop folder contains our plop config. Plop is a code generator that
    allows us to generate boilerplate code for our components. Please see the{" "}
    <a href="/?path=/docs/styleguide-4-plop--default-story">Plop Page</a> docs
    for more information.
  </p>
  <h2>public/</h2>
  <hr />
  <p>The public/ folder contains all our static assets.</p>
  <h2>.husky</h2>
  <hr />
  <p>
    The .husky folder contains our husky config. Husky is a git hook manager
    that allows us to run scripts before or after git commands.
  </p>
</Story>
