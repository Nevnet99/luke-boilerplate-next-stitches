  <h2>Nextjs</h2>
  <hr />
  <p>
    Nextjs is a React framework that allows us to build server-side rendered
    applications. It is a great tool for SEO and performance. It also allows us
    to use serverless functions to handle API calls.
  </p>
  <h2>Husky</h2>
  <hr />
  <p>
    Husky is a tool that allows us to run scripts before git commits and pushes.
    We use it to run linting and testing scripts before pushing to the remote
    repository.
  </p>
  <h2>Styled Components</h2>
  <hr />
  <p>
    Styled Components is a library that allows us to write CSS in our React
    components. It allows us to write CSS in a way that is familiar to us and
    also allows us to use props to dynamically change styles.
  </p>
  <h2>Plop</h2>
  <hr />
  <p>
    Plop is a tool that allows us to generate boilerplate code. We use it to
    generate components and pages. To use plop you can run the following command
    in the terminal:
    <code>yarn generate</code>
  </p>
  <h2>Vitest</h2>
  <hr />
  <p>
    Vitest is a testing framework that allows us to write tests in a way that is
    familiar to us. It is a great tool for testing React components.
  </p>
  <h2>React Testing Library</h2>
  <hr />
  <p>
    React Testing Library is a testing library that allows us to write tests in
    a way that is familiar to us. It is a great tool for testing React
    components.
  </p>
  <h2>Typescript</h2>
  <hr />
  <p>
    Typescript is a superset of Javascript that allows us to write code with
    types and implement type safety before runtime. It is a great tool for
    writing code that is easier to maintain and less prone to bugs.
  </p>
  <h2>Storybook</h2>
  <hr />
  <p>
    Storybook is a tool that allows us to write documentation for our React
    components and develop our components in isolation. It is a great tool for
    writing documentation and building components that is easy to maintain and
    less prone to bugs.
  </p>
  <p>
    I'd suggest adding Chromatic to your Storybook project to get visual
    regression testing for free. It's a great tool for catching bugs before they
    get merged into the main branch.
  </p>
  <h2>Eslint</h2>
  <hr />
  <p>
    Eslint is a tool that allows us to lint our code. It's a great tool for
    keeping our code consitant and clean.
  </p>
</Story>
